# APA Coach v1.2 BYOK LLM Guide

## What This Feature Is

v1.2 adds optional LLM-enhanced feedback for the Check module. It is not enabled by default and it is not a hosted AI service.

BYOK means "bring your own key." A reviewer may provide an OpenAI-compatible `base URL`, `API key`, and `model` in the browser.

## What Gets Sent

The app does not save student text. However, when a user clicks `Enhance with my LLM`, the current citation text is sent from the browser to the configured provider.

That means:

- APA Coach does not store the text.
- GitHub Pages does not store the text.
- The browser sends the text to the provider configured by the reviewer.
- The provider's own privacy policy and account settings determine how that provider handles the request.

## Where the API Key Is Stored

The API key is stored in browser `sessionStorage`.

This means:

- It may remain after a page refresh.
- It is normally cleared when the tab or browser session closes.
- It is not written to the GitHub repository.
- It is not written to a project server, because v1.2 has no backend server.

This is still browser-side key handling, so it is not recommended for open student use.

## Why This Is Not Recommended for Public Student Use

v1.2 is still a static website without a secure backend proxy.

If students enter their own API keys:

- The key exists in the student's browser environment.
- The key owner is responsible for cost.
- The provider may block direct browser requests through CORS.
- The course team cannot centrally manage cost, privacy, rate limits, or logs.

This feature is best suited for controlled faculty review or private demonstration.

## What a CORS Failure Means

Some LLM providers do not allow browser pages to call their APIs directly. In that case, the app may show a CORS or network error.

This is not an APA citation-rule error. It is a browser security restriction.

Possible options:

- Use an OpenAI-compatible provider that allows browser requests.
- Use a compatible proxy that you control.
- Move a future version to a secure backend proxy where the key is stored server-side.

v1.2 does not include a proxy service.

## Current LLM Feedback Boundaries

LLM feedback is tutoring support only. It does not:

- Verify DOI, URL, author, date, or source truth.
- Guarantee final APA correctness.
- Replace instructor policy.
- Reliably convert MLA or Chicago into APA.
- Override rule-based diagnosis.

If the LLM returns an unsafe or invalid structure, the app displays a safe fallback instead of showing unvalidated output as final advice.

## Recommended Faculty Review Flow

1. Use rule-based Check first without configuring an LLM.
2. Paste or select the MLA sample and confirm that the app identifies it as closer to MLA than APA.
3. Read the privacy notice in LLM Settings.
4. If testing LLM enhancement, use a personal test key and low-risk sample text.
5. Confirm whether course policy allows citation text to be sent to an external provider.

Do not move into a public student pilot until course policy, privacy language, cost control, and provider settings have been reviewed.

# 中文附录：APA Coach v1.2 BYOK LLM 使用说明

## 这是什么

v1.2 给 Check 模块增加了可选的 LLM 增强反馈。它不是默认开启功能，也不是项目托管的 AI 服务。

BYOK 指 bring your own key。评审者可以在浏览器里提供 OpenAI-compatible API 的 `base URL`、`API key` 和 `model`。

## 会发送什么

本项目不保存学生文本。但是，当用户点击 `Enhance with my LLM` 时，当前 citation 文本会从浏览器发送到用户配置的 provider。

也就是说：APA Coach 不保存文本，GitHub Pages 不保存文本，但 provider 如何处理请求取决于 provider 自己的隐私政策和账号设置。

## API key 存在哪里

API key 存在浏览器 `sessionStorage`。刷新页面后可能保留，关闭标签页或浏览器会话后通常清除。它不会写入 GitHub 仓库，也不会写入项目服务器，因为 v1.2 没有后端。

但这仍然是浏览器端 key 处理方式，不建议开放给学生长期使用。

## 为什么不建议公开学生使用

v1.2 仍是静态网站，没有安全后端代理。如果学生输入自己的 API key，费用、隐私、速率限制和 provider 日志都难以由课程方统一管理。有些 provider 也会因为 CORS 阻止浏览器直接请求。

## 当前边界

LLM 反馈只作为 tutoring support。它不验证 DOI、URL、作者、年份或来源真实性，不保证最终 APA 完全正确，不替代 instructor policy，也不会可靠地把 MLA/Chicago 自动转换成 APA。

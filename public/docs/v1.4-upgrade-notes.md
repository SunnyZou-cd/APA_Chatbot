# APA Coach v1.4 Upgrade Notes

## Purpose

v1.4 focuses on Document Check. It addresses the main v1.3 acceptance gaps: DOCX import reliability, rich-text preservation, broader APA citation detection, and more useful correction generation.

## Main Changes

- Document Check is now the main APA review surface.
- The old Check workflow is available inside Document Check as Single reference / excerpt mode.
- DOCX/PDF/TXT parsing runs through `/api/document-check` on Vercel.
- DOCX parsing uses server-side Mammoth HTML conversion instead of a frontend `mammoth.browser` chunk.
- Rich text pasted from Word or Google Docs remains visible in the editable review area.
- PDF extraction returns text, while layout-sensitive formatting remains manual review.

## APA Analysis Improvements

The parser now recognizes:

- `(Smith, 2024)`
- `(Smith & Lee, 2024)`
- `(Smith et al., 2024)`
- `(Smith, 2024; Lee, 2022)`
- `Smith (2024)`
- `Smith and Lee (2024)`

Reference parsing now creates candidate fields for author, year, title, source/container, volume/issue/pages, DOI/URL, guessed source type, and confidence.

## Correction Behavior

Corrections are now source-type-specific:

- Journal articles italicize the journal title and volume.
- Books, reports, and webpages italicize the standalone work title.
- Chapter-like entries ask for source metadata when fields are insufficient.

Every correction includes confidence and a reminder to confirm source type and metadata. If the app cannot parse enough fields, it says that a reliable correction cannot be generated yet and lists missing fields.

## Privacy Boundary

v1.4 does not add Google login, a database, or persistent file storage. Uploaded files are parsed for the current request only. The default Document Check workflow does not call an LLM.

## Remaining Manual Review Items

The app still does not claim final validation for:

- Title page layout.
- Font and double spacing.
- Margins.
- Hanging indents.
- APA heading levels.
- PDF italics/font extraction.

These are shown as manual review items because they are difficult to verify reliably from extracted text.

## Recommended Faculty Review Question

Does v1.4 make Document Check useful enough as a professor-review prototype while clearly separating automated citation/reference feedback from layout-sensitive manual review?

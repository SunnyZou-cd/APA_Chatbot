# APA Coach v1.0 功能性测试与分析报告

测试日期：2026-06-15  
测试对象：APA Coach v1.0  
GitHub Pages 地址：https://sunnyzou-cd.github.io/APA_Chatbot/  
GitHub 仓库：https://github.com/SunnyZou-cd/APA_Chatbot  

## 1. 测试结论

当前 v1.0 已达到“教授可打开、可初步评审、可作为学习型 APA 原型演示”的交付标准。核心页面可以加载，Build / Check / Learn / Faculty Review 四个入口均可使用，GitHub Pages 部署成功，构建和高危依赖审计通过。

但 v1.0 仍应被定义为教学原型，而不是可靠的 APA citation engine。它适合展示产品方向、交互流程和学术诚信边界，不适合直接承诺“APA 格式完全正确”或“可用于真实作业最终检查”。

## 4. 测试中发现的问题

### P1：GitHub Pages 下 Faculty Review 文档链接使用根路径

现象：

- `https://sunnyzou-cd.github.io/docs/professor-handoff.md` 返回 404。
- 正确路径应为 `https://sunnyzou-cd.github.io/APA_Chatbot/docs/professor-handoff.md`。

原因：

- v1.0 的 Faculty Review 链接写成 `/docs/...`，在 GitHub Pages 子路径部署时会跳出仓库路径。

处理：

- 已将链接改为基于 `import.meta.env.BASE_URL` 的自适应路径。
- 已补充 `src/vite-env.d.ts`，让 TypeScript 识别 Vite 环境变量。

### P2：Check 模块存在旧版本文案

现象：

- “No obvious v0.0 issues were found.”

处理：

- 已改为 “No obvious v1.0 issues were found.”

### P2：Build 模块 fallback 可能让学生误以为系统自动补全了来源信息

现象：

- 当 author/title/source 为空时，输出会使用示例 fallback。
- 虽然页面会显示 warning，但 reference entry 仍可能看起来像一个完整 citation。

建议：

- v1.1 应改成更明确的 incomplete state，不在 citation output 里静默混入示例作者或示例标题。

### P2：Citation formatting 仍是演示级，不是 APA 7 完整规则引擎

现象：

- source type 覆盖面有限。
- 对多人作者、组织作者、缺失作者、网页日期格式、书章 editor 格式、DOI 标准化等支持不足。

建议：

- v1.1 应接入 Citation.js / CSL 或至少建立更完整的 deterministic formatter。

### P3：Practice mode 只有正确答案反馈，没有解释

现象：

- 学生选择答案后，只能看到视觉状态。
- 没有解释为什么对/错。

建议：

- v1.1 增加 explanation 和 retry feedback。

## 5. 当前能力边界建议

建议对教授和学生明确说明：

当前版本可以：

- 演示 APA Coach 的产品定位和教学流程。
- 生成少量 source type 的 citation pair 示例。
- 提供基础 rule-based diagnosis。
- 通过 hint-first/reveal-second 方式支持学习。
- 展示 APA rule cards、paper setup checklist 和 AI citation/disclosure 提醒。
- 作为 faculty review prototype 使用。

当前版本不应该承诺：

- 不保证所有 APA 7 citation 格式完全正确。
- 不保证能识别所有 citation 错误。
- 不适合作为最终作业提交前的唯一检查工具。
- 不验证 DOI、URL、作者、日期或来源真实性。
- 不进行全文论文格式检查。
- 不保存学生输入，也不提供教师 analytics。
- 不接入真实 AI 模型，不提供语义层面的写作诊断。

建议面向教授的表述：

> APA Coach v1.0 is a learning-first prototype for faculty review. It demonstrates citation-building, hint-first feedback, APA rule cards, and academic-integrity boundaries. It should not yet be treated as a complete APA validator or a final submission checker.

## 6. v1.0 是否适合给教授评审

适合，但建议用“prototype review”而不是“pilot-ready tool”定位。

推荐教授评审重点：

- 教学边界是否清楚。
- 是否符合 Psychology / APA writing 课程需求。
- hint-first 设计是否能减少直接复制答案。
- APA rule card 解释是否足够清楚。
- 是否需要课程政策或 instructor-specific rule cards。

不建议现在进入真实学生 pilot，除非先完成：

- 更可靠 citation formatter。
- 更完整测试集。
- 明确隐私/数据政策。
- 教师认可的 APA examples。
- 课程政策与 AI disclosure 文案审查。

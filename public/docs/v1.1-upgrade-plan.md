# APA Coach v1.1 升级计划书

## 1. 升级目标

v1.1 的目标是把当前 v1.0 从“可演示原型”推进到“可进行小范围教师评审和受控课堂试用准备”的版本。

核心方向：

- 提高 citation 输出可靠性。
- 减少 fallback 造成的误解。
- 增强 Check 模块诊断质量。
- 让 Learn 模块更像学习工具，而不是静态材料。
- 增加测试用例和质量门槛。

v1.1 仍不建议直接作为全校公开工具，也不建议接入真实学生数据分析。

## 2. 产品定位

推荐继续使用：

```text
APA Coach
```

v1.1 的一句话定位：

```text
A learning-first APA citation and writing-format coach that helps students understand what to fix and why.
```

v1.1 仍应避免：

- APA Generator
- Paper Writer
- Essay Rewriter
- Plagiarism Checker
- Final APA Validator

## 3. v1.1 重点升级范围

### 3.1 Build：更可靠的 citation builder

升级内容：

- 移除 author/title/source 的静默示例 fallback。
- 当关键字段缺失时，reference output 显示 `Incomplete citation` 状态。
- 对每个 source type 定义 required fields 和 optional fields。
- 增加 field-level validation，例如：
  - author missing
  - date missing
  - title missing
  - source/container missing
  - DOI/URL format warning
- 引入至少 12 个 curated examples：
  - journal article with DOI
  - journal article without DOI
  - book
  - edited book chapter
  - webpage with organization author
  - webpage with no date
  - report
  - YouTube/video source
  - course material
  - generative AI tool
  - source with missing author
  - source with missing date

验收标准：

- 空字段不会生成看似完整的 citation。
- 每种 source type 至少有一个正确示例和一个缺失字段示例。
- 所有 warning 都能对应一个 rule card 或 field helper。

### 3.2 Check：从 regex demo 升级到 structured diagnosis

升级内容：

- 将当前 regex-only diagnosis 拆成规则模块：
  - author-date rule
  - title capitalization rule
  - DOI/URL rule
  - missing page/volume issue rule
  - reference/in-text mismatch rule
- 每条 diagnosis 输出：
  - issue
  - why it matters
  - hint
  - student action
  - revealable suggested correction
  - confidence
  - rule source
- 增加 false-positive 保护：
  - 不确定时使用 “Check this with your instructor/source”。
  - 避免把所有大写词都判断为 title case 错误。

验收标准：

- 至少 20 条 curated incorrect citation cases。
- 每条 case 有 expected issues。
- Check 输出不应把示例 correction 当作真实 source verification。

### 3.3 Learn：从静态卡片升级到学习闭环

升级内容：

- Practice prompts 增加 explanation。
- 错误答案显示 why not。
- 每个 rule card 增加：
  - common mistake
  - student action
  - “ask instructor when...” 提醒
- 增加 Learn topic filter：
  - In-text citations
  - References
  - Paper setup
  - AI citation/disclosure

验收标准：

- 学生答题后能看到解释，而不只是颜色反馈。
- 每个 rule card 至少有一个 common mistake。
- AI disclosure 模块明确区分 APA citation 与 course policy。

### 3.4 Faculty Review：更适合教授评审

升级内容：

- 增加 faculty review checklist：
  - pedagogy fit
  - academic integrity fit
  - APA accuracy concerns
  - privacy concerns
  - pilot readiness
- 增加 sample walkthrough。
- 增加 “not pilot-ready until...” 清单。
- 增加 v1.0 test report 和 v1.1 plan 链接。

验收标准：

- 教授不需要读仓库也能理解当前能力边界。
- Faculty Review 页面能直接说明哪些功能是 prototype，哪些功能需要后续验证。

### 3.5 测试与质量门槛

升级内容：

- 新增本地测试脚本：
  - citation formatter unit tests
  - diagnosis rule tests
  - source type required-field tests
- 新增 CI 检查：
  - `npm run build:pages`
  - `npm audit --audit-level=high`
  - unit tests
- 增加 documented test matrix。

验收标准：

- 至少 30 个 unit test cases。
- GitHub Actions 失败时不发布 Pages。
- 每次发布前测试报告可复用。

## 4. 明确不进入 v1.1 的内容

v1.1 不做：

- 真实 AI API 接入。
- 学生账号系统。
- 数据库。
- 教师 dashboard。
- 学生文本保存。
- 全文论文上传和解析。
- 自动改写段落。
- DOI 在线验证。

这些内容可以进入 v1.2 或 v2.0，但需要隐私、成本、学术诚信和 faculty policy 先行设计。

## 5. 建议技术方案

### 5.1 Citation formatting

短期方案：

- 保留 deterministic formatter。
- 用 source-type templates + validation rules 明确控制输出。

中期方案：

- 评估 Citation.js / CSL。
- 对比 APA 官方例子和 Purdue OWL 例子。
- 建立 expected output snapshot。

### 5.2 Diagnosis rules

推荐结构：

```text
src/lib/diagnosis/
  authorDateRule.ts
  titleCaseRule.ts
  doiUrlRule.ts
  sourceTypeClassifier.ts
  testCases.ts
```

每条 rule 应返回统一结构：

```text
severity
message
whyItMatters
hint
studentAction
suggestedCorrection
confidence
ruleId
```

### 5.3 Test data

推荐新增：

```text
src/data/examples.ts
src/data/testCases.ts
```

数据应区分：

- correct examples
- incorrect examples
- incomplete examples
- uncertain examples

交付：

### Milestone 4：Faculty Review Pack

交付：

### Milestone 5：Quality Gate

交付：

- unit tests
- CI test step
- updated deployment guide
- updated professor handoff

## 7. 推荐优先级

| Priority | Item | Reason |
| :-- | :-- | :-- |
| P0 | Fix incomplete citation behavior | Prevents misleading students. |
| P0 | Add deterministic validation per source type | Required for trustworthy coaching. |
| P1 | Expand diagnosis test cases | Makes Check credible. |
| P1 | Add practice explanations | Improves learning value. |
| P1 | Link test report and v1.1 plan in Faculty Review | Helps professor review. |
| P2 | Citation.js / CSL evaluation | Useful but should follow test-case design. |
| P2 | More polished visual states | Good for adoption, less urgent than correctness. |

## 8. Suggested v1.1 Success Criteria

v1.1 can be considered complete when:

- GitHub Pages deployment is green.
- No high-severity npm audit issues.
- At least 30 automated rule/formatter tests pass.
- Build supports all source types without silent sample fallback.
- Check mode returns structured feedback for at least 20 known incorrect citations.
- Faculty Review clearly states capability boundaries and pilot readiness limits.
- Professor can use the site and documents without reading the codebase.
